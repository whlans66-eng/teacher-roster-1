# 📱 啟用iPhone掃碼功能 - GitHub Pages 部署指南

## 🎯 為什麼需要部署到GitHub Pages？

**問題**：iPhone Safari瀏覽器出於安全考慮，不允許本地HTML檔案（`file://`）存取攝影機

**解決方案**：將系統部署到HTTPS網站，這樣iPhone就可以使用攝影機掃碼了！

GitHub Pages提供**免費的HTTPS託管**，完美解決這個問題！✨

---

## 📋 設定步驟（只需3分鐘）

### 步驟 1️⃣：合併PR

1. 訪問這個連結建立PR：
   ```
   https://github.com/whlans66-eng/teacher-roster-1/pull/new/claude/github-pages-setup-011CUtAbU6oraYyEmnPRJY55
   ```

2. 點擊 **"Create pull request"** 按鈕

3. 再點擊 **"Merge pull request"** 按鈕

4. 確認合併

### 步驟 2️⃣：啟用GitHub Pages

1. 在您的GitHub儲存庫頁面，點擊 **Settings**（設定）標籤

2. 在左側選單找到 **"Pages"** 選項

3. 在 **"Source"** (來源) 部分：
   - **Branch**: 選擇 `main`
   - **Folder**: 選擇 `/ (root)`

4. 點擊 **"Save"** (儲存) 按鈕

5. 等待 1-2 分鐘，頁面會顯示您的網站位址：
   ```
   https://whlans66-eng.github.io/teacher-roster-1/
   ```

### 步驟 3️⃣：在iPhone上訪問

1. 在iPhone Safari中開啟：
   ```
   https://whlans66-eng.github.io/teacher-roster-1/
   ```

2. 點擊進入 **"資產管理系統"**

3. 點擊右上角 **"📷 掃描條碼"** 按鈕

4. 允許Safari存取攝影機

5. 🎉 **現在您的iPhone可以掃碼了！**

## 📱 新增到iPhone主畫面（推薦）

讓系統像APP一樣使用：

1. 在Safari中開啟資產管理系統

2. 點擊底部的 **"分享"** 按鈕（方框帶箭頭的圖示）

3. 向下滾動，選擇 **"加入主畫面"**

4. 給它取個名字，比如 **"資產管理"**

5. 點擊 **"新增"**

6. 現在主畫面上有一個圖示，點擊就能直接開啟！

## 🔧 如果遇到問題

### 問題1：找不到Pages設定
**解決**：確保您有儲存庫的管理員權限

### 問題2：網站顯示404
**解決**：
1. 確認PR已經合併到main分支
2. 等待1-2分鐘讓GitHub部署完成
3. 重新整理頁面重試

### 問題3：iPhone還是不能掃碼
**解決**：
1. 確認使用的是HTTPS位址（不是HTTP）
2. 確認已經允許Safari存取攝影機
3. 在Safari設定中檢查攝影機權限

### 問題4：攝影機權限被拒絕
**解決**：
1. 開啟iPhone **設定**
2. 找到 **Safari**
3. 找到 **攝影機**
4. 選擇 **"詢問"** 或 **"允許"**
