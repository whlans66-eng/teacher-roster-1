# 资产管理系统使用说明

## 系统概述

这是一个功能完整的**资产管理系统**，专门用于管理设备资产（投影仪、电脑、平板等）。系统支持资产登记、借用归还、维修记录、盘点管理、报废处理等全流程管理，并集成了条码扫描功能。

## 🚀 快速开始

### 打开系统
直接在浏览器中打开 `asset-management.html` 文件即可使用。

### 浏览器要求
- 推荐使用 Chrome、Edge、Firefox 等现代浏览器
- 如需使用摄像头扫码功能，需要 HTTPS 环境或本地文件访问

## 📋 核心功能

### 1. 资产登记
- **新增资产**：点击右上角"➕ 新增资产"按钮
- **资产信息**：
  - 资产编号（自动生成或手动输入）
  - 资产名称（如：投影仪）
  - 资产分类（电子设备、办公设备、家具、其他）
  - 条码号（支持扫描）
  - 购买日期
  - 购买价格
  - 存放位置
  - 资产描述

### 2. 条码扫描 📷
系统支持**三种扫码方式**：

#### 方式一：摄像头扫描
1. 点击"📷 扫描条码"按钮
2. 允许浏览器访问摄像头
3. 将条码对准摄像头即可自动识别

#### 方式二：扫码枪输入
- 直接使用扫码枪扫描，系统会自动识别并跳转到资产详情
- 或在"扫描条码"弹窗中的输入框使用扫码枪

#### 方式三：手动输入
- 在"扫描条码"弹窗中手动输入条码号或资产编号

**支持的条码格式**：
- Code 128
- EAN-13 / EAN-8
- Code 39
- UPC

### 3. 资产借用/归还 📤📥

#### 借用流程
1. 切换到"📤 借用/归还"标签
2. 填写借用信息：
   - 资产编号（可扫码）
   - 借用人姓名
   - 借用日期
   - 预计归还日期
   - 备注说明
3. 点击"确认借用"

#### 归还流程
1. 在"已借出资产"列表中找到对应资产
2. 点击"归还"按钮
3. 系统自动记录归还日期和时长

### 4. 维修记录 🔧
1. 切换到"🔧 维修记录"标签
2. 填写维修信息：
   - 资产编号（可扫码）
   - 维修日期
   - 故障描述
   - 维修方案
   - 维修费用
   - 维修状态（待维修/维修中/已完成）
3. 提交后自动更新资产状态

### 5. 资产盘点 📋
1. 点击右上角"📋 盘点"按钮
2. 使用扫码枪或摄像头扫描资产条码
3. 每扫描一个资产，自动添加到盘点列表
4. 盘点完成后，点击"导出报告"生成盘点报告（TXT格式）

**盘点报告包含**：
- 盘点日期
- 盘点数量
- 资产清单（编号、名称、分类、状态、盘点时间）

### 6. 报废管理 🗑️
1. 切换到"🗑️ 报废管理"标签
2. 填写报废信息：
   - 资产编号（可扫码）
   - 报废日期
   - 报废原因
3. 确认后资产状态变为"已报废"
4. 报废资产不再参与统计和借用

### 7. 使用历史 📊
- 自动记录所有借用归还历史
- 支持按资产编号、借用人搜索
- 显示借用时长和归还状态

### 8. 统计分析 📈
系统提供4种可视化图表：

1. **资产状态分布**（饼图）
   - 可用资产数量
   - 已借出资产数量
   - 维修中资产数量
   - 已报废资产数量

2. **资产分类统计**（柱状图）
   - 各分类资产数量对比

3. **月度借用趋势**（折线图）
   - 最近6个月的借用趋势分析

4. **维修费用统计**（柱状图）
   - 各资产累计维修费用

## 📊 仪表盘说明

系统顶部显示4个关键指标：
- **总资产数**：系统中所有资产总数
- **可用资产**：当前可借用的资产数量
- **已借出**：当前借出中的资产数量
- **维修中**：正在维修的资产数量

## 🔍 资产搜索与筛选

在"📦 资产列表"标签中：
- **关键词搜索**：支持按资产名称、编号、条码搜索
- **分类筛选**：按电子设备、办公设备、家具、其他筛选
- **状态筛选**：按可用、已借出、维修中、已报废筛选

## 💾 数据存储

系统使用浏览器的 `localStorage` 存储数据：
- 所有数据保存在本地浏览器中
- 关闭浏览器后数据不会丢失
- 清除浏览器数据会导致系统数据丢失

**数据备份建议**：
- 定期使用盘点功能导出资产报告
- 建议定期手动备份浏览器数据

## 🎯 使用场景示例

### 场景1：新设备入库
1. 收到新投影仪后，点击"➕ 新增资产"
2. 粘贴条码标签到设备上
3. 使用扫码功能录入条码号
4. 填写设备信息（名称、价格、位置等）
5. 保存完成

### 场景2：设备借出
1. 用户来借投影仪
2. 点击"📤 借用/归还"标签
3. 扫描设备条码
4. 填写借用人信息和预计归还日期
5. 确认借用

### 场景3：设备归还
1. 用户归还设备
2. 在"已借出资产"列表中找到该设备
3. 点击"归还"按钮
4. 系统自动记录归还时间

### 场景4：定期盘点
1. 每月进行资产盘点
2. 点击"📋 盘点"按钮
3. 逐个扫描所有资产条码
4. 盘点完成后导出报告
5. 对比报告发现是否有资产遗失

### 场景5：设备维修
1. 发现投影仪故障
2. 切换到"🔧 维修记录"标签
3. 扫描设备条码
4. 填写故障描述和维修信息
5. 设备状态自动更新为"维修中"

## 🛠️ 技术特性

- **纯前端实现**：无需服务器，直接打开即用
- **响应式设计**：支持电脑、平板、手机访问
- **现代化UI**：使用 Tailwind CSS，界面美观
- **数据可视化**：集成 Chart.js 图表库
- **条码识别**：集成 QuaggaJS 条码扫描库
- **本地存储**：使用 localStorage 持久化数据

## 📱 移动端使用

系统支持在手机/平板上使用：
1. 使用手机浏览器打开 `asset-management.html`
2. 添加到主屏幕，像APP一样使用
3. 手机摄像头可直接扫描条码

## ⚠️ 注意事项

1. **条码标准化**：
   - 建议使用统一的条码格式
   - 条码应与资产编号关联

2. **权限管理**：
   - 系统无登录功能，建议在受控环境使用
   - 如需权限管理，建议部署到服务器并添加认证

3. **数据安全**：
   - 定期备份数据
   - 不建议在公共设备上使用

4. **浏览器兼容**：
   - 摄像头扫码需要 HTTPS 或 localhost
   - 部分旧版浏览器可能不支持

## 🎨 界面定制

如需修改界面颜色或样式，可编辑 HTML 文件中的 CSS：
- `.stat-gradient-1` 到 `.stat-gradient-4`：统计卡片颜色
- `.gradient-bg`：背景渐变色
- `.modern-btn`：按钮样式

## 📞 技术支持

如有问题或建议，请：
1. 检查浏览器控制台是否有错误信息
2. 确认浏览器版本是否支持
3. 尝试清除缓存后重新打开
