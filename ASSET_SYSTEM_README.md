# 資產管理系統使用說明

## 系統概述

這是一個功能完整的**資產管理系統**，專門用於管理設備資產（投影機、電腦、平板等）。系統支援資產登記、借用歸還、維修記錄、盤點管理、報廢處理等全流程管理，並整合了條碼掃描功能。

## 🚀 快速開始

### 開啟系統
直接在瀏覽器中開啟 `asset-management.html` 檔案即可使用。

### 瀏覽器需求
- 推薦使用 Chrome、Edge、Firefox 等現代瀏覽器
- 如需使用攝影機掃碼功能，需要 HTTPS 環境或本地檔案存取

## 📋 核心功能

### 1. 資產登記
- **新增資產**：點擊右上角"➕ 新增資產"按鈕
- **資產資訊**：
  - 資產編號（自動產生或手動輸入）
  - 資產名稱（如：投影機）
  - 資產分類（電子設備、辦公設備、家具、其他）
  - 條碼號（支援掃描）
  - 購買日期
  - 購買價格
  - 存放位置
  - 資產描述

### 2. 條碼掃描 📷
系統支援**三種掃碼方式**：

#### 方式一：攝影機掃描
1. 點擊"📷 掃描條碼"按鈕
2. 允許瀏覽器存取攝影機
3. 將條碼對準攝影機即可自動辨識

#### 方式二：掃碼槍輸入
- 直接使用掃碼槍掃描，系統會自動辨識並跳轉到資產詳情
- 或在"掃描條碼"彈窗中的輸入框使用掃碼槍

#### 方式三：手動輸入
- 在"掃描條碼"彈窗中手動輸入條碼號或資產編號

**支援的條碼格式**：
- Code 128
- EAN-13 / EAN-8
- Code 39
- UPC

### 3. 資產借用/歸還 📤📥

#### 借用流程
1. 切換到"📤 借用/歸還"標籤
2. 填寫借用資訊：
   - 資產編號（可掃碼）
   - 借用人姓名
   - 借用日期
   - 預計歸還日期
   - 備註說明
3. 點擊"確認借用"

#### 歸還流程
1. 在"已借出資產"清單中找到對應資產
2. 點擊"歸還"按鈕
3. 系統自動記錄歸還日期和時長

### 4. 維修記錄 🔧
1. 切換到"🔧 維修記錄"標籤
2. 填寫維修資訊：
   - 資產編號（可掃碼）
   - 維修日期
   - 故障描述
   - 維修方案
   - 維修費用
   - 維修狀態（待維修/維修中/已完成）
3. 提交後自動更新資產狀態

### 5. 資產盤點 📋
1. 點擊右上角"📋 盤點"按鈕
2. 使用掃碼槍或攝影機掃描資產條碼
3. 每掃描一個資產，自動新增到盤點清單
4. 盤點完成後，點擊"匯出報告"產生盤點報告（TXT格式）

**盤點報告包含**：
- 盤點日期
- 盤點數量
- 資產清單（編號、名稱、分類、狀態、盤點時間）

### 6. 報廢管理 🗑️
1. 切換到"🗑️ 報廢管理"標籤
2. 填寫報廢資訊：
   - 資產編號（可掃碼）
   - 報廢日期
   - 報廢原因
3. 確認後資產狀態變為"已報廢"
4. 報廢資產不再參與統計和借用

### 7. 使用歷史 📊
- 自動記錄所有借用歸還歷史
- 支援按資產編號、借用人搜尋
- 顯示借用時長和歸還狀態

### 8. 統計分析 📈
系統提供4種視覺化圖表：

1. **資產狀態分布**（圓餅圖）
   - 可用資產數量
   - 已借出資產數量
   - 維修中資產數量
   - 已報廢資產數量

2. **資產分類統計**（柱狀圖）
   - 各分類資產數量對比

3. **月度借用趨勢**（折線圖）
   - 最近6個月的借用趨勢分析

4. **維修費用統計**（柱狀圖）
   - 各資產累計維修費用

## 📊 儀表板說明

系統頂部顯示4個關鍵指標：
- **總資產數**：系統中所有資產總數
- **可用資產**：當前可借用的資產數量
- **已借出**：當前借出中的資產數量
- **維修中**：正在維修的資產數量

## 🔍 資產搜尋與篩選

在"📦 資產清單"標籤中：
- **關鍵詞搜尋**：支援按資產名稱、編號、條碼搜尋
- **分類篩選**：按電子設備、辦公設備、家具、其他篩選
- **狀態篩選**：按可用、已借出、維修中、已報廢篩選

## 💾 資料儲存

系統使用瀏覽器的 `localStorage` 儲存資料：
- 所有資料儲存在本地瀏覽器中
- 關閉瀏覽器後資料不會遺失
- 清除瀏覽器資料會導致系統資料遺失

**資料備份建議**：
- 定期使用盤點功能匯出資產報告
- 建議定期手動備份瀏覽器資料

## 🎯 使用場景範例

### 場景1：新設備入庫
1. 收到新投影機後，點擊"➕ 新增資產"
2. 黏貼條碼標籤到設備上
3. 使用掃碼功能錄入條碼號
4. 填寫設備資訊（名稱、價格、位置等）
5. 儲存完成

### 場景2：設備借出
1. 使用者來借投影機
2. 點擊"📤 借用/歸還"標籤
3. 掃描設備條碼
4. 填寫借用人資訊和預計歸還日期
5. 確認借用

### 場景3：設備歸還
1. 使用者歸還設備
2. 在"已借出資產"清單中找到該設備
3. 點擊"歸還"按鈕
4. 系統自動記錄歸還時間

### 場景4：定期盤點
1. 每月進行資產盤點
2. 點擊"📋 盤點"按鈕
3. 逐個掃描所有資產條碼
4. 盤點完成後匯出報告
5. 對比報告發現是否有資產遺失

### 場景5：設備維修
1. 發現投影機故障
2. 切換到"🔧 維修記錄"標籤
3. 掃描設備條碼
4. 填寫故障描述和維修資訊
5. 設備狀態自動更新為"維修中"

## 🛠️ 技術特性

- **純前端實現**：無需伺服器，直接開啟即用
- **響應式設計**：支援電腦、平板、手機存取
- **現代化UI**：使用 Tailwind CSS，介面美觀
- **資料視覺化**：整合 Chart.js 圖表庫
- **條碼辨識**：整合 QuaggaJS 條碼掃描庫
- **本地儲存**：使用 localStorage 持久化資料

## 📱 移動端使用

系統支援在手機/平板上使用：
1. 使用手機瀏覽器開啟 `asset-management.html`
2. 新增到主畫面，像APP一樣使用
3. 手機攝影機可直接掃描條碼

## ⚠️ 注意事項

1. **條碼標準化**：
   - 建議使用統一的條碼格式
   - 條碼應與資產編號關聯

2. **權限管理**：
   - 系統無登入功能，建議在受控環境使用
   - 如需權限管理，建議部署到伺服器並新增認證

3. **資料安全**：
   - 定期備份資料
   - 不建議在公共設備上使用

4. **瀏覽器相容**：
   - 攝影機掃碼需要 HTTPS 或 localhost
   - 部分舊版瀏覽器可能不支援

## 🎨 介面客製化

如需修改介面顏色或樣式，可編輯 HTML 檔案中的 CSS：
- `.stat-gradient-1` 到 `.stat-gradient-4`：統計卡片顏色
- `.gradient-bg`：背景漸層色
- `.modern-btn`：按鈕樣式
